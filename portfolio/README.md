# Gerrit Düthler — portfolio + private application tracker

One Streamlit page (`app.py`) with **two tabs**:

| Tab | What | Who sees it |
| --- | ---- | ----------- |
| 📄 **Portfolio** | Public, bilingual (NL/EN) one-pager + CV download | Everyone |
| 🔒 **Privé** | Application tracker + live vacancy search | You only — unlocks with `PRIVATE_PASSWORD` |

`tracker.py` is the same private area as a standalone local app, if you prefer it
in its own window. The shared logic lives in `private.py`.

## Edit your content
All portfolio text is in **`content.py`** — replace anything between `‹ ›`. Add a
square photo as `assets/photo.jpg` (optional). The clean CV PDF (`assets/cv.pdf`,
no address/phone/birth date) is generated by `python generate_cv.py`.

## Settings (`.streamlit/secrets.toml`, git-ignored)
Copy `.streamlit/secrets.toml.example` and fill in:
- `PRIVATE_PASSWORD` — unlocks the **🔒 Privé** tab (set a strong one before deploying publicly).
- `ADZUNA_APP_ID` / `ADZUNA_APP_KEY` — better NL vacancy results (free key at developer.adzuna.com).
- `PORTFOLIO_URL` — link from the private overview back to your public site.

## Run locally
```bash
pip install -r requirements.txt
streamlit run app.py        # portfolio (public) + Privé tab (password)
streamlit run tracker.py    # optional: the private area on its own
```

## Privacy
- The tracker **data** lives in `data/applications.xlsx`, which is **git-ignored** — never
  committed or uploaded. It opens straight in Excel.
- The **🔒 Privé** tab only unlocks with `PRIVATE_PASSWORD`; the portfolio tab stays public.
- ⚠️ On Streamlit Community Cloud the filesystem is **temporary** — tracker edits there won't
  persist and the data file isn't deployed. For durable tracking, run it **locally**; use the
  **📱 Mobiel overzicht (HTML)** download to carry a snapshot on your phone.

## Hosting
- **Static portfolio** (fastest, always-on): the generated `docs/` site via GitHub Pages
  (`generate_site.py`). Settings → Pages → `main` → `/docs`.
- **Full app** (portfolio + private area): deploy `app.py` to Streamlit Community Cloud and
  set `PRIVATE_PASSWORD` in the app's secrets.
